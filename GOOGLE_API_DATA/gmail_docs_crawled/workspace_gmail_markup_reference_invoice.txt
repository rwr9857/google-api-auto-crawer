사용 사례
다음 사용 사례는 Invoice 스키마의 일반적인 예를 보여줍니다. 있습니다. 다음 예를 사용하여 마크업이 올바르게 구성되었는지 확인하세요.
참고: 시작하기 전에 이메일에 스키마를 삽입 [https://developers.google.com/workspace/gmail/markup/embedding-schemas-in-emails?hl=ko]하는 방법을 이해하고 테스트 스키마 [https://developers.google.com/workspace/gmail/markup/testing-your-schema?hl=ko]를 숙지해야 합니다.
자동 결제 인보이스
다음은 인보이스 유형을 사용한 마크업의 예입니다. 이 경우 자동 결제가 지정되었습니다.
--- 탭: JSON-LD [https://developers.google.com/workspace/gmail/markup/reference/invoice?hl=ko#json-ld] ---
<script type='application/ld+json'>
{
  "@context": "http://schema.org",
  "@type": "Invoice",
  "accountId": "123-456-789",
  "minimumPaymentDue": {
    "@type": "PriceSpecification",
    "price": "$70.00"
  },
  "paymentDue": "2015-11-22T08:00:00+00:00",
  "paymentStatus": "PaymentAutomaticallyApplied",
  "provider": {
    "@type": "Organization",
    "name": "Mountain View Utilities"
  },
  "totalPaymentDue": {
    "@type": "PriceSpecification",
    "price": "$70.00"
  }
}
</script>

마크업 테스트
이메일 마크업 테스터 도구 [https://www.google.com/webmasters/markup-tester/?hl=ko]를 사용하여 마크업의 유효성을 검사할 수 있습니다. 마크업 코드를 붙여넣고 확인 버튼을 클릭하여 콘텐츠를 스캔하면 오류에 관한 보고서를 받게 됩니다.
사양
이메일 세부정보를 검토하여 다음과 같은 추가 속성이 있는지 확인하세요. 인보이스에 적용됩니다. 이러한 추가 속성을 마크업하면 Google에서 인보이스에 대해 더 자세한 설명을 표시합니다.
인보이스
유형 이름: 인보이스 [https://developers.google.com/workspace/gmail/markup/reference/types/Invoice?hl=ko]
무형 [https://developers.google.com/workspace/gmail/markup/reference/types/Intangible?hl=ko] 확장
이름 유형 설명
accountId 텍스트 [https://developers.google.com/workspace/gmail/markup/reference/types/Text?hl=ko] 결제가 적용될 계정의 식별자입니다.
billingPeriod 소요 시간 [https://developers.google.com/workspace/gmail/markup/reference/types/Duration?hl=ko] 인보이스를 계산하는 데 사용되는 시간 간격입니다.
중개인 Organization [https://developers.google.com/workspace/gmail/markup/reference/types/Organization?hl=ko] 또는 Person [https://developers.google.com/workspace/gmail/markup/reference/types/Person?hl=ko] 구매자와 판매자 간의 교환을 주선하는 법인입니다. 대부분의 경우 중개인은 교환과 관련된 제품 또는 서비스의 소유권을 획득하거나 양도하지 않습니다. 법인이 중개인인지, 판매자인지, 구매자인지 명확하지 않은 경우에는 나머지 두 가지 조건 중 하나가 우선 적용됩니다.
카테고리 PhysicalActivityCategory [https://developers.google.com/workspace/gmail/markup/reference/types/PhysicalActivityCategory?hl=ko], Text [https://developers.google.com/workspace/gmail/markup/reference/types/Text?hl=ko] 또는 Thing [https://developers.google.com/workspace/gmail/markup/reference/types/Thing?hl=ko] 항목의 카테고리입니다. 큰 부호나 슬래시는 카테고리 계층 구조를 비공식적으로 나타내기 위해 사용할 수 있습니다.
confirmationNumber 텍스트 [https://developers.google.com/workspace/gmail/markup/reference/types/Text?hl=ko] 지정된 주문 또는 결제 금액이 수신되었음을 확인하는 번호입니다.
고객 Organization [https://developers.google.com/workspace/gmail/markup/reference/types/Organization?hl=ko] 또는 Person [https://developers.google.com/workspace/gmail/markup/reference/types/Person?hl=ko] 주문 또는 인보이스 지불 당사자
minimumPaymentDue PriceSpecification [https://developers.google.com/workspace/gmail/markup/reference/types/PriceSpecification?hl=ko] 현재 요구되는 최소 결제액입니다.
paymentDue DateTime [https://developers.google.com/workspace/gmail/markup/reference/types/DateTime?hl=ko] 결제 기한입니다.
paymentMethod PaymentMethod [https://developers.google.com/workspace/gmail/markup/reference/types/PaymentMethod?hl=ko] 주문에 사용한 신용카드 또는 기타 결제 수단의 이름입니다.
paymentMethodId 텍스트 [https://developers.google.com/workspace/gmail/markup/reference/types/Text?hl=ko] 사용된 결제 수단의 식별자입니다 (예: 신용카드 마지막 4자리).
paymentStatus 텍스트 [https://developers.google.com/workspace/gmail/markup/reference/types/Text?hl=ko] 결제 상태 인보이스 결제 여부
provider Organization [https://developers.google.com/workspace/gmail/markup/reference/types/Organization?hl=ko] 또는 Person [https://developers.google.com/workspace/gmail/markup/reference/types/Person?hl=ko] 예약을 제공하는 조직입니다.
referencesOrder 항목 [https://developers.google.com/workspace/gmail/markup/reference/types/Order?hl=ko] 이 인보이스와 관련된 주문입니다. 하나 이상의 주문이 하나의 인보이스로 통합될 수 있습니다.
scheduledPaymentDate 날짜 [https://developers.google.com/workspace/gmail/markup/reference/types/Date?hl=ko] 인보이스 결제가 예약된 날짜입니다.
totalPaymentDue PriceSpecification [https://developers.google.com/workspace/gmail/markup/reference/types/PriceSpecification?hl=ko] 미결제 총액입니다.
참고: Google에서 사용하는 일부 스키마는 아직 schema.org [http://schema.org/]의 표준화 프로세스를 진행 중이므로 향후 변경될 수 있습니다. 자세히 알아보기 [https://developers.google.com/workspace/gmail/markup/reference/schema-org-proposals?hl=ko]
