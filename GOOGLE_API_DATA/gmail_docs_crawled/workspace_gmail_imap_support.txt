Gmail IMAP팀은 다양한 플랫폼을 함께 사용하여 지원을 제공하고 있습니다. 아래에서 내게 가장 적합한 지원 옵션이 무엇인지 알아보세요.
IMAP 클라이언트 관련 문제
IMAP을 설정하려고 하거나 IMAP 클라이언트에 문제가 있는 사용자이고 IMAP 코드를 작성하려는 개발자가 아닌 경우 먼저 Gmail 고객센터 [https://support.google.com/mail/answer/78892?hl=ko]를 살펴보거나 Gmail 제품 포럼 [https://support.google.com/mail/community/?hl=ko]에 질문하는 것이 좋습니다.
질문
Google은 널리 사용되는 Q&A 웹사이트인 Stack Overflow [https://stackoverflow.com/questions/tagged/gmail-imap]에서 기술적인 질문을 해결해 드립니다. 이 사이트는 Google에서 운영하지 않지만 Google 계정을 사용하여 로그인 [https://stackoverflow.com/users/login]할 수 있습니다.
Stack Overflow에는 다양한 주제에 대한 질문이 포함되어 있으며, Gmail IMAP와 관련된 질문은 gmail-imap 태그로 표시됩니다. 질문에 추가 태그를 포함하여 관련 기술 전문가의 주의를 끌 수 있습니다.
새로운 질문을 처음으로 올리기 전에 잠시 시간을 내어 Stack Overflow FAQ [https://stackoverflow.com/tour]를 읽어보세요. 사이트와 커뮤니티에서 질문에 대한 답변을 받으려면 가이드라인과 팁을 준수해야 합니다.
기존 질문 검색 새 질문하기 [https://stackoverflow.com/questions/ask?tags=gmail-imap]
문제 및 오류
API에서 오류가 발생하는 경우 우선 위의 설명대로 Stack Overflow에서 검색하거나 질문을 올려보세요. 문제가 발생한 위치를 최대한 추적하여 해당 코드 섹션만 올리시기 바랍니다.
버그
문제의 원인이 API 자체의 버그로 판명되었다면 Issue Tracker [https://issuetracker.google.com/bookmark-groups/78025?hl=ko]를 통해 신고할 수 있습니다. 버그 설명에 다음 정보를 포함해야 합니다.
문제에 대한 설명 및 정상일 때의 작동 방식
문제를 재현하는 데 사용할 수 있는 단계 목록 또는 소규모 샘플 코드 스니펫
개발 환경에 대한 정보(프로그래밍 언어, 라이브러리 버전 등)
기존 버그 검색 새 버그 신고하기 [https://issuetracker.google.com/issues/new?component=191609&template=1394715&hl=ko]
누락된 기능
Gmail IMAP 구현에 추가하고 싶은 기능이 누락된 경우 문제 추적기 [https://issuetracker.google.com/bookmark-groups/78025?hl=ko]에 기능 요청을 제출할 수 있습니다. 추가되었으면 하는 구체적인 기능과 그 이유를 설명하세요. 가능하다면 사용 사례 및 해당 기능이 가져올 이점을 자세히 서술하시기 바랍니다.
