등록 가이드라인
사용자에게 동적 이메일을 발송할 준비가 되면 사용하는 각 발신자 이메일 주소를 Google에 등록해야 합니다. 이메일 콘텐츠가 변경되거나 향후 동일한 발신자 이메일 주소가 다른 사용 사례에 사용되더라도 이메일을 한 번만 등록하면 됩니다.
발신자 이메일을 등록하려면 다음 단계를 완료하세요.
아래에 나열된 모든 가이드라인 및 요구사항 [https://developers.google.com/workspace/gmail/ampemail/register?hl=ko#registration_guidelines]을 충족해야 합니다.
프로덕션 서버(또는 유사한 SPF/DKIM/DMARC/From:/Return-Path: 헤더가 있는 서버)에서 발송된 실제 프로덕션용 이메일을 동적 이메일 MIME 부분을 포함하여 ampforemail.whitelisting@gmail.com로 전송합니다. 모든 가이드라인 및 요구사항 [https://developers.google.com/workspace/gmail/ampemail/register?hl=ko#registration_guidelines]을 준수하는지 확인하기 위해 필요한 정보입니다. 이 이메일을 보낼 때 다음 사항을 따르세요.
이메일을 보내기 전에 Gmail에서 테스트 [https://developers.google.com/workspace/gmail/ampemail/testing-dynamic-email?hl=ko]하여 이메일의 AMP 버전이 올바른지 확인합니다.
이메일을 전달하는 대신 직접 보냅니다. Gmail에서는 이메일을 전달할 때 AMP MIME 부분을 삭제합니다.
주의: 검토를 위해 이메일을 보내지 않거나 테스트 (빈) 이메일을 보내거나 AMP MIME 부분이 포함되지 않은 이메일을 보내면 Google에서 등록 신청을 거부합니다. 메일링 리스트에 ampforemail.whitelisting@gmail.com를 추가하지 마세요. AMP MIME 부분이 포함되지 않은 정적 HTML 이메일이 많이 전송될 수 있습니다. 검토 시 Google은 등록하려는 발신자 이메일 주소에서 가장 최근에 받은 이메일만 엽니다.
등록 양식 [https://forms.gle/fDqLE1gh4wMB6BeQ6]을 작성하고 답변을 기다립니다. 대부분의 요청은 영업일 기준 5일 이내에 응답하도록 노력하고 있습니다.
동적 이메일 발신자로 참여하려면 Gmail 사용자에게 보낸 메일이 차단되거나 스팸으로 분류되는 것을 방지하기 [https://support.google.com/mail/answer/81126?hl=ko] 가이드라인을 준수해야 하며, 이 가이드라인은 예고 없이 변경될 수 있습니다. 현재 가이드라인을 준수할 책임은 개발자에게 있습니다. 악용사례 또는 정책 위반이 발견되거나 Google의 재량에 따라 언제든지 참여가 취소될 수 있습니다. Google에서는 규정 준수를 확인하기 위해 등록 절차 중 또는 등록이 완료된 후에 추가 정보를 요청할 수 있습니다.
등록 가이드라인
Google에 등록하려면 다음 가이드라인을 충족해야 합니다.
데모 또는 'Hello World' 예시가 아닌 실제 프로덕션 품질의 이메일 예시를 보내야 합니다. 제목이나 본문에 {name} 및 {count}와 같은 자리표시자 문자열이나 [Test], [Proof mail], [Sample]와 같은 테스트 문자열을 넣지 마세요.
이메일에 유사한 HTML 또는 텍스트 MIME 부분이 있어야 합니다. 사용자가 동적 이메일을 사용 중지한 경우와 같이 대신 이 이메일이 표시되는 경우가 많습니다.
이메일에 버그가 없어야 합니다.
보안 요구사항 [https://developers.google.com/workspace/gmail/ampemail/security-requirements?hl=ko]에 설명된 대로 SPF, DKIM, DMARC를 사용하여 이메일을 인증해야 합니다.
이메일용 AMP 권장사항을 검토하고 이메일이 권장사항을 준수하는지 확인합니다.
등록은 발신자 이메일 주소별로 작동합니다. 발신자 이메일 주소를 두 개 이상 등록하려면 각 주소를 별도로 등록하고 자체 제작 품질의 이메일을 전송해야 합니다.
등록은 발신자 이메일 도메인별로 작동하지 않으므로 별도의 이메일 주소가 동일한 도메인을 사용하더라도 동일하게 적용됩니다.
사용자 또는 일반인과 같은 서드 파티가 이메일을 직접 작성하고 전송하도록 허용해서는 안 됩니다. 보내는 이메일에 게시물이나 댓글과 같은 사용자 제작 콘텐츠가 포함될 수 있습니다. 테스트 도구 및 '이메일 플레이그라운드'는 허용 목록에 추가할 수 없으며 대신 Gmail에서 AMP 이메일 테스트 [https://developers.google.com/workspace/gmail/ampemail/testing-dynamic-email?hl=ko]에 설명된 단계를 따라야 합니다.
이메일은 Gmail 사용자에게 보낸 메일이 차단되거나 스팸으로 분류되는 것을 방지하기 [https://support.google.com/mail/answer/81126?hl=ko] 가이드라인을 준수해야 합니다.
사용자의 스팸 신고 비율이 낮습니다.
등록할 발신자 이메일 주소의 eTLD+1 [https://developer.mozilla.org/en-US/docs/Glossary/eTLD]은 발신자에게 속한 유효한 웹사이트 (예: 주차된 도메인이 아님)를 게재하거나 리디렉션해야 합니다. 예를 들어 발신자 이메일 주소가 noreply@mail.example.com이면 https://example.com에서 웹사이트를 제공해야 합니다. IP 필터 없이 전 세계에서 웹사이트에 액세스할 수 있어야 합니다.
