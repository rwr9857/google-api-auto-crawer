다음 동영상에서는 애플리케이션의 REST API, 이메일 마크업, 부가기능과 같은 다양한 Gmail 개발자 기능을 학습하는 데 도움이 되는 개념을 설명합니다. 각 동영상에서는 일반적으로 짧은 샘플 앱을 안내하여 특정 기능을 빠르게 시작할 수 있도록 지원합니다.
Gmail API 소개
Google에서 Gmail API를 만든 이유는 무엇인가요? 이 동영상에서는 개발자에게 이메일에 대한 프로그래매틱 액세스의 역사와 Gmail용 API가 생성된 동기를 설명합니다. Gmail API를 사용하면 개발자가 표준 이메일 프로토콜로 할 수 있는 것 이상을 할 수 있습니다.
(러닝 타임: 5분 42초)
Gmail API를 통한 이메일 액세스 혁신
Gmail API를 검토한 후 이 동영상에서는 채팅 이메일 대화목록 (메시지가 3개 이상인 대화목록)의 제목을 표시하는 중간 수준의 예를 살펴봅니다. 시청자는 코드 둘러보기를 볼 뿐만 아니라 미국 역사 보너스 수업도 들을 수 있습니다.
(재생 시간: 8:32)
Gmail API로 서명 수정하기
Gmail API를 사용하면 개발자가 사용자 설정을 수정하는 앱을 만들 수 있습니다. 이 동영상에서는 사용자 서명 [https://developers.google.com/workspace/gmail/api/guides/alias_and_signature_settings?hl=ko#managing_signatures]을 수정하는 예를 안내합니다. API로 수정할 수 있는 기타 설정에는 이메일 전달, 부재중 자동응답, IMAP/POP를 통한 외부 액세스, 필터가 포함됩니다.
(러닝타임: 6:01)
이 기능은 블로그 게시물 'Gmail API로 이메일 서명 수정하기' [https://gsuite-developers.googleblog.com/2016/12/modifying-email-signatures-with-the-gmail-api.html]에서도 설명합니다.
이메일 마크업으로 뛰어난 환경 만들기
개발자는 마크업을 사용하여 이메일에 작업을 추가하여 사용자 참여도를 높일 수 있습니다. 이 동영상에서는 이미 전송 중인 메시지에 이메일 마크업 [https://developers.google.com/workspace/gmail/markup?hl=ko]을 추가하여 고객 경험을 개선하는 방법을 보여줍니다.
(실행 시간: 4:41)
Gmail 부가기능으로 비용 보고서 신속 처리하기
부가기능을 사용하면 Gmail을 확장하고 내 앱의 기능을 Gmail의 사용자 인터페이스에 통합할 수 있습니다. 이 동영상에서는 ExpenseIt! Gmail 부가기능 [https://github.com/googleworkspace/gmail-add-on-codelab]: 직장인이 받은편지함에서 바로 영수증을 스프레드시트로 처리하여 경비 보고서를 작성할 수 있는 단일 공간을 만드는 데 도움이 되는 앱입니다. 해당 codelab [http://g.co/codelabs/gmail-add-ons]을 사용하여 부가기능을 단계별로 빌드합니다.
(러닝타임: 5:18)
이 기능은 블로그 게시물 '이제 모든 개발자가 Gmail 부가기능 프레임워크를 사용할 수 있습니다' [https://developers.googleblog.com/en/gmail-add-ons-framework-now-available-to-all-developers/]에서도 설명합니다.
