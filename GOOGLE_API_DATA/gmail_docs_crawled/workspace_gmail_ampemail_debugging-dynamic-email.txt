대체 이유
Gmail에서는 동적 이메일과 관련된 일반적인 문제를 디버그하는 기능을 제공합니다.
Gmail 웹에서 대체 디버깅 배너 사용 설정
개발자는 디버깅 배너를 사용 설정하여 이메일이 AMP 이메일로 렌더링되지 않는 이유를 확인할 수 있습니다. 배너에는 이메일의 AMP 부분이 렌더링되지 않고 그 이유가 표시됩니다. 각 이유에 관한 자세한 내용은 다음 섹션을 참고하세요.
특정 이메일에 디버깅 배너를 사용 설정하려면 개발자 설정에서 발신자 주소를 허용 목록에 추가한 다음 페이지를 새로고침해야 합니다. 자세한 내용은 Gmail에서 AMP 이메일 테스트 [https://developers.google.com/workspace/gmail/ampemail/testing-dynamic-email?hl=ko]를 참고하세요.
참고: 설정 패널에서 동적 이메일을 사용 중지한 경우 배너가 표시되지 않습니다. 배너가 표시되지 않으면 설정을 사용 설정했는지 확인하고 페이지를 새로고침합니다.
대체 이유
다음은 동적 이메일이 렌더링되지 않고 기본 텍스트 또는 HTML로 대체되는 일반적인 이유입니다.
대체 이유
ACCOUNT_TYPE 사용 중인 계정 유형에서 동적 이메일이 지원되지 않습니다. 새 테스트 Gmail 계정을 사용하여 이메일을 확인해 보세요.
AUTH_FAILED 이메일이 인증 요구사항 [https://developers.google.com/workspace/gmail/ampemail/security-requirements?hl=ko#sender_authentication] 중 하나를 충족하지 못했습니다.
AUTO_FORWARDED 다른 계정에서 이메일을 자동으로 전달 [https://support.google.com/mail/answer/10957?hl=ko]했습니다. 이메일에서 AMP 콘텐츠를 렌더링하려면 이메일이 전송된 계정으로 이메일을 열어야 합니다.
BROWSER_ERROR 예상치 못한 문제가 발생했습니다. 지원되는 브라우저 [https://developers.google.com/workspace/gmail/ampemail/supported-browsers?hl=ko] 중 하나를 사용하고 있는지 확인하고 다시 시도하세요.
DKIM_FAILED 이메일이 DKIM (도메인 키 확인 메일) 인증에 실패했습니다. 자세한 내용은 인증 요구사항 [https://developers.google.com/workspace/gmail/ampemail/security-requirements?hl=ko#sender_authentication]을 참고하세요.
DKIM_NOT_MATCHING_FROM 이메일이 DKIM (Domain Keys Identified Mail) 조정 [https://developers.google.com/workspace/gmail/ampemail/security-requirements?hl=ko#dkim_alignment]에 실패했습니다.
DYNAMIC_EMAIL_DISABLED 동적 이메일이 사용 중지되었습니다. Gmail 설정 패널에서 동적 이메일이 사용 설정되어 있는지 확인합니다.
HIDING_IMAGES 항상 외부 이미지 표시 설정이 사용 중지됩니다. 설정 패널에서 이 설정을 다시 사용 설정하고 페이지를 새로고침합니다.
INTERNAL_ERROR Gmail에서 예상치 못한 문제가 발생했습니다.
INVALID_AMP AMP가 잘못되었습니다. 자세한 내용은 이메일용 AMP 사양을 참고하고 AMP 이메일 유효성 검사 단계 [https://amp.dev/documentation/guides-and-tutorials/learn/validation-workflow/validate_emails/?format=email]를 따르세요.
MALFORMED 이메일에 text/x-amp-html 부분이 두 개 이상 포함되어 있거나 대체 text/html 또는 text/plain 부분이 없습니다. 자세한 내용은 전송 요구사항 [https://developers.google.com/workspace/gmail/ampemail/testing-dynamic-email?hl=ko#delivery_requirements]을 참고하세요.
MESSAGE_CLIPPED 이메일의 text/x-amp-html 부분이 너무 깁니다. 자세한 내용은 전송 요구사항 [https://developers.google.com/workspace/gmail/ampemail/testing-dynamic-email?hl=ko]을 참고하세요.
NOT_AVAILABLE_ON_PLATFORM 발신자가 이 플랫폼에서 동적 콘텐츠를 사용 중지했습니다. 웹 클라이언트를 사용하는 경우 모바일에서 이메일을 확인해 보세요.
NOT_MAIN_WINDOW 이메일이 새 창에서 열렸습니다. 기본 Gmail 창에서 이 이메일을 열어보세요.
OFFLINE 클라이언트가 오프라인 상태입니다. 인터넷 연결을 확인하고 다시 시도해 보세요.
OLD_EMAIL 이메일이 30일 전에 전송되었습니다.
PHISHY 이메일이 피싱으로 표시되었습니다. 이메일이 보안 요구사항 및 이메일 권장사항 [https://support.google.com/mail/answer/81126?hl=ko]을 준수하는지 확인합니다.
REFRESH_REQUIRED 클라이언트를 새로고침해야 합니다. 웹 클라이언트를 사용 중이고 오프라인 모드를 사용 설정하지 않은 경우 페이지를 정상적으로 새로고침합니다. 오프라인 모드를 사용 설정한 경우 강제로 새로고침해야 합니다. 모바일 앱을 사용하는 경우 앱 데이터를 삭제하고 앱을 다시 실행해 보세요.
SPAM 이메일이 스팸으로 표시되었습니다. 이메일이 보안 요구사항 및 이메일 권장사항 [https://support.google.com/mail/answer/81126?hl=ko]을 준수하는지 확인합니다.
SPF_FAILED 이메일이 SPF (Sender Policy Framework) 인증에 실패했습니다. 자세한 내용은 인증 요구사항 [https://developers.google.com/workspace/gmail/ampemail/security-requirements?hl=ko#sender_authentication]을 참고하세요.
SUSPICIOUS 이메일이 의심스러워 보입니다. 이메일이 보안 요구사항 및 이메일 권장사항 [https://support.google.com/mail/answer/81126?hl=ko]을 준수하는지 확인합니다.
THREAD_TOO_LONG 답장 체인이 너무 길어졌습니다. 동적 콘텐츠는 대화목록의 마지막 10개 메시지에 대해서만 렌더링됩니다.
TIMEOUT 동적 콘텐츠를 로드하는 데 시간이 너무 오래 걸렸습니다. 인터넷 연결을 확인하고 다시 시도해 보세요.
TLS_ENCRYPTION 이메일이 TLS로 암호화되지 않았습니다. 자세한 내용은 TLS 보안 요구사항 [https://developers.google.com/workspace/gmail/ampemail/security-requirements?hl=ko#tls_encryption]을 참고하고 TLS를 사용하여 이메일을 다시 보내세요.
TRANSLATED 이메일에 번역이 사용 설정되어 있습니다. Gmail에서 자동 번역을 사용 중지하고 다시 시도해 보세요.
WRONG_VERSION 기기에서 동적 콘텐츠가 지원되지 않습니다. 자세한 내용은 지원되는 브라우저 [https://developers.google.com/workspace/gmail/ampemail/supported-browsers?hl=ko] 목록을 참고하세요.
