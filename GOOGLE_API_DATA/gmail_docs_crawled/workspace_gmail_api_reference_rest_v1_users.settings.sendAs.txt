리소스: SendAs
'다음으로 보내기' 별칭과 연결된 설정으로, 계정과 연결된 기본 로그인 주소 또는 맞춤 '보낸사람' 주소일 수 있습니다. 보내기 별칭은 웹 인터페이스의 '다른 주소에서 메일 보내기' [https://support.google.com/mail/answer/22370?hl=ko] 기능에 해당합니다.
JSON 표현
{
  "sendAsEmail": string,
  "displayName": string,
  "replyToAddress": string,
  "signature": string,
  "isPrimary": boolean,
  "isDefault": boolean,
  "treatAsAlias": boolean,
  "smtpMsa": {
    object (SmtpMsa [https://developers.google.com/workspace/gmail/api/reference/rest/v1/users.settings.sendAs?hl=ko#SmtpMsa])
  },
  "verificationStatus": enum (VerificationStatus [https://developers.google.com/workspace/gmail/api/reference/rest/v1/users.settings.sendAs?hl=ko#VerificationStatus])
}
필드
sendAsEmail
string
이 별칭을 사용하여 전송된 메일의 '보낸사람:' 헤더에 표시되는 이메일 주소입니다. 생성을 제외한 모든 작업에 대해 읽기 전용입니다.
displayName
string
이 별칭을 사용하여 전송된 메일의 '보낸사람:' 헤더에 표시되는 이름입니다. 맞춤 '보낸사람' 주소가 비어 있으면 Gmail에서 계정과 연결된 기본 주소에 사용된 이름으로 '보낸사람:' 헤더를 채웁니다. 관리자가 사용자가 이름 형식을 업데이트할 수 있는 기능을 사용 중지한 경우 기본 로그인에 대해 이 필드를 업데이트하라는 요청이 자동으로 실패합니다.
replyToAddress
string
이 별칭을 사용하여 전송된 메일의 'Reply-To:' 헤더에 포함되는 선택사항 이메일 주소입니다. 이 필드가 비어 있으면 Gmail에서 'Reply-To:' 헤더를 생성하지 않습니다.
signature
string
Gmail 웹 UI에서 이 별칭으로 작성된 메일에 포함되는 선택사항인 HTML 서명입니다. 이 서명은 새 이메일에만 추가됩니다.
isPrimary
boolean
이 주소가 계정에 로그인하는 데 사용되는 기본 주소인지 여부입니다. 모든 Gmail 계정에는 기본 주소가 정확히 하나 있으며, 이 주소는 발신자 별칭 모음에서 삭제할 수 없습니다. 이 필드는 읽기 전용입니다.
isDefault
boolean
새 메시지를 작성하거나 부재중 자동 답장을 보낼 때 이 주소가 기본 '보낸사람:' 주소로 선택되는지 여부입니다. 모든 Gmail 계정에는 기본 발신자 주소가 정확히 하나 있으므로 클라이언트가 이 필드에 쓸 수 있는 유효한 값은 true뿐입니다. 주소의 이 값을 false에서 true로 변경하면 다른 이전 기본 주소의 이 입력란이 false이 됩니다.
treatAsAlias
boolean
Gmail에서 이 주소를 사용자의 기본 이메일 주소의 별칭으로 처리 [https://support.google.com/a/answer/1710338?hl=ko]해야 하는지 여부입니다. 이 설정은 맞춤 '보낸사람' 별칭에만 적용됩니다.
smtpMsa
object (SmtpMsa [https://developers.google.com/workspace/gmail/api/reference/rest/v1/users.settings.sendAs?hl=ko#SmtpMsa])
이 별칭을 사용하여 전송된 메일의 발신 릴레이로 사용되는 선택적 SMTP 서비스입니다. 이 필드가 비어 있으면 발신 메일이 Gmail 서버에서 대상 SMTP 서비스로 직접 전송됩니다. 이 설정은 맞춤 '보낸사람' 별칭에만 적용됩니다.
verificationStatus
enum (VerificationStatus [https://developers.google.com/workspace/gmail/api/reference/rest/v1/users.settings.sendAs?hl=ko#VerificationStatus])
이 주소가 발신자 별칭으로 사용되도록 확인되었는지 여부를 나타냅니다. 읽기 전용입니다. 이 설정은 맞춤 '보낸사람' 별칭에만 적용됩니다.
SmtpMsa
SMTP 서비스와의 통신을 위한 구성입니다.
JSON 표현
{
  "host": string,
  "port": integer,
  "username": string,
  "password": string,
  "securityMode": enum (SecurityMode [https://developers.google.com/workspace/gmail/api/reference/rest/v1/users.settings.sendAs?hl=ko#SecurityMode])
}
필드
host
string
SMTP 서비스의 호스트 이름입니다. 필수 항목입니다.
port
integer
SMTP 서비스의 포트입니다. 필수 항목입니다.
username
string
SMTP 서비스 인증에 사용되는 사용자 이름입니다. 이 필드는 SendAs 설정을 생성하거나 업데이트하는 요청에 지정할 수 있는 쓰기 전용 필드이며 응답에는 채워지지 않습니다.
password
string
SMTP 서비스 인증에 사용할 비밀번호입니다. 이 필드는 SendAs 설정을 생성하거나 업데이트하는 요청에 지정할 수 있는 쓰기 전용 필드이며 응답에는 채워지지 않습니다.
securityMode
enum (SecurityMode [https://developers.google.com/workspace/gmail/api/reference/rest/v1/users.settings.sendAs?hl=ko#SecurityMode])
SMTP 서비스와의 통신을 보호하는 데 사용되는 프로토콜입니다. 필수 항목입니다.
SecurityMode
SMTP 서비스와의 통신을 보호하는 데 사용할 수 있는 프로토콜입니다.
열거형
securityModeUnspecified 지정되지 않은 보안 모드입니다.
none 원격 SMTP 서비스와의 통신이 안전하지 않습니다. 포트 25가 필요합니다.
ssl 원격 SMTP 서비스와의 통신은 SSL을 사용하여 보호됩니다.
starttls 원격 SMTP 서비스와의 통신은 STARTTLS를 사용하여 보호됩니다.
VerificationStatus
주소의 소유권이 발신자 별칭으로 사용되기 위해 확인되었는지 여부를 나타냅니다.
열거형
verificationStatusUnspecified 지정되지 않은 인증 상태입니다.
accepted 이제 이 주소를 발신자 별칭으로 사용할 수 있습니다.
pending 주소가 소유자의 확인을 기다리는 중입니다.
