이메일 작성의 어려움
이메일을 작성할 때 문제가 있는 경우 다음 섹션이 도움이 될 수 있습니다.
코드를 head에 넣을 수 없음
이메일 구성 방식에 따라 이메일 본문의 할인/제품을 담당하는 시스템은 HTML 헤더를 처리하는 시스템과 별개입니다 (헤더에 삽입할 할인 정보를 알지 못할 수 있음). 거래 정보를 처리하는 시스템 부분에서 전체 스크립트 태그 또는 거래 <script>{"@type": "DiscountOffer", …}</script>가 포함된 부분을 본문에 삽입할 수 있도록 이메일 본문에 스크립트 태그의 일부 또는 전체를 삽입하세요.
이메일 서비스 제공업체에서 스크립트 태그를 삭제함
ESP에서 예외를 만들지 않는 경우 허용될 수 있는 주석을 다는 microdata 방식이 Schema.org에 있습니다.
<head><script>...</script></head> 대신 마이크로데이터 예는 HTML의 어느 곳에나 있을 수 있으며 다음과 같이 표시됩니다.
 <div itemscope itemtype="http://schema.org/Organization">
      <meta itemprop="logo" content="https://www.gstatic.com/images/branding/product/1x/googleg_48dp.png" />
    </div>
    <div itemscope itemtype="http://schema.org/DiscountOffer">
      <meta itemprop="description" content="20% off" />
      <meta itemprop="discountCode" content="PROMO" />
      <meta itemprop="availabilityStarts" content="2018-01-01T08:00:00-07:00" />
      <meta itemprop="availabilityEnds" content="2018-04-30T23:59:59-07:00" />
    </div>
    <div itemscope itemtype="http://schema.org/PromotionCard">
      <meta itemprop="image" content="https://www.google.com/gmail-for-marketers/promo-tab/markup-tool/sample.png" />
    </div>
이 대체 방법은 미리보기 도구 [https://developers.google.com/workspace/gmail/promotab/overview?hl=ko]에서도 테스트할 수 있습니다.
더 쉽게 구현할 수 있도록 스크립트 태그 분리
오래된 콘텐츠를 전송하지 않거나 콘텐츠 블록에서 기능이 제공되는 경우 주석을 분할할 수 있습니다. 스크립트 태그의 일부를 헤드에 배치하고 나머지는 본문의 블록에 배치합니다.
다음은 로고와 대체 제목만 헤드에 배치하고 배지, 만료일, 주석의 이미지 미리보기는 이메일 본문에 배치하는 예입니다.
<head>
<script type="application/ld+json">
  [{
    "@context": "http://schema.org/",
    "@type": "Organization",
    "logo": "https://www.gstatic.com/images/branding/product/1x/googleg_48dp.png"
  },{
     "@context": "http://schema.org/",
     "@type": "EmailMessage",
     // An alternative subject line shown when deal badge or discount code is also displayed.
     "subjectLine": "Alternate Subject Line"
  }]
</script>
<body>
<script type="application/ld+json">
  [{
    "@context": "http://schema.org/",
    "@type": "DiscountOffer",
    "description": "20% Off",
    "discountCode": "PROMO",
    "availabilityStarts": "2019-01-01T08:00:00-07:00",
    "availabilityEnds": "2019-12-30T23:59:59-07:00"
  },{
     "@context": "http://schema.org/",
     "@type": "PromotionCard",
     "image": "https://www.google.com/gmail-for-marketers/promo-tab/markup-tool/sample.png"
  }]
</script>
참고: 주석 필드를 헤드와 본문 사이에 배포할 수 없습니다. 예를 들어 할인 혜택("@type": "DiscountOffer")의 특가 배지와 프로모션 코드를 본문에 배치하면서 만료일을 헤드에 배치할 수는 없습니다. 이 경우 주석의 불확실한 부분만 하나 표시됩니다.
이메일 확인 또는 테스트 문제
다음 섹션에서는 이메일을 보거나 테스트하려고 할 때 발생하는 일반적인 문제를 설명합니다.
이메일 번들이 전혀 표시되지 않음
최신 버전의 Gmail 앱이 설치된 기기를 사용하고 있는지 확인합니다. 프로모션 탭에서 화면을 아래로 당겨 이메일을 새로고침합니다.
기기를 다시 시작해 보세요.
아직도 문제가 해결되지 않았나요? 태블릿과 같은 다른 기기에서 확인해 보세요. 태블릿의 Gmail 버전이 다를 수 있으며 번들이 다르게 표시될 수 있습니다.
소비자에게만 이메일 번들이 표시됩니다. 이 환경은 사용자에게 영향을 미치지 않습니다.
참고: 이메일이 홀드백 그룹에 포함될 수 있습니다. 위의 방법이 모두 작동하지 않으면 promotabtesting@gmail.com로 끝나는 새 계정을 만드세요. 예를 들어 mycompanypromotabtesting@gmail.com를 만들고 테스트에 새 계정을 사용합니다.
이메일이 프로모션 탭으로 이동하지 않음
이메일이 평소에 마케팅 이메일을 보내는 하위 도메인에서 전송되었는지 확인합니다. 잘 알려지지 않은 하위 도메인에서 전송하면 Gmail 탭 분류기가 혼동되어 이메일이 프로모션 탭에 배치되지 않을 수 있습니다.
계정에 이메일을 기본 탭으로 전송하는 이메일 필터가 없는지 확인합니다.
이메일이 올바른 탭에 표시되도록 별도의 발신자 하위 도메인에서 전송합니다. Gmail 분류기에 관한 자세한 내용은 대량 발신자 가이드라인 [https://support.google.com/mail/answer/81126?hl=ko]을 참고하세요.
promotabtesting@gmail.com 계정을 사용하는 경우 첫 번째 이메일의 동작은 의도된 것입니다. 이메일을 프로모션 탭으로 드래그하고 다른 제목 줄을 사용하여 동일한 발신자 주소에서 테스트 계정으로 이메일을 보냅니다. 이제 시각화할 수 있습니다.
번들은 표시되지만 번들에 브랜드의 주석이 달린 이메일이 표시되지 않음
이메일이 Gmail 프로모션 탭에 있는지 확인합니다.
이메일이 아직 열리지 않았고 전날 전송되지 않았는지 확인합니다.
번들 내 이메일을 한 기기에서 보면 다른 기기의 번들에 해당 이메일이 다시 채워지지 않습니다. 미리 보려면 새 테스트 이메일을 보내야 합니다.
여러 기기에서 시각화하려면 테스트 계정을 여러 개 만드는 것이 좋습니다. 팀원 중 한 명이 이메일을 열어 나머지 팀원의 번들에 이메일이 표시되지 않을 수 있습니다.
전송된 이메일의 코드에 주석이 올바른지 확인합니다 (시작일 전이거나 만료된 경우 이메일이 표시되지 않을 수 있음).
최신 버전의 Gmail 앱을 사용하고 있는지 확인합니다. 태블릿과 같이 다른 기기에서 확인해 보세요. 태블릿에는 다른 버전의 Gmail 앱이 설치되어 있어 이메일이 다르게 표시될 수 있습니다.
Gmail 프로모션 탭 화면을 아래로 당겨 Gmail 앱을 새로고침합니다.
기기를 다시 시작해 보세요.
Gmail 프로모션 탭에서는 민감한 카테고리 (예: 성인용 콘텐츠 또는 채권 추심)를 사용할 수 없습니다.
이미지가 주석에 포함되어 있을 때 이미지 미리보기가 표시되지 않음
스크립트 태그를 확인하여 전송된 이미지에 올바른 URL이 있는지 확인합니다.
Gmail 프로모션 탭 화면을 아래로 당겨 Gmail 앱을 새로고침합니다.
이미지가 민감한 주제로 간주되는 경우 Gmail 프로모션 탭에 이미지가 표시되지 않을 수 있습니다 (예: 성인용 콘텐츠 또는 채권 추심).
한 기기에는 번들이 표시되지만 다른 기기에는 표시되지 않음
동일한 이메일 계정으로 보고 있는지 확인합니다.
이메일을 한 기기에서 먼저 열면 순위가 변경되어 다른 기기에서는 번들로 표시되지 않을 수 있습니다.
