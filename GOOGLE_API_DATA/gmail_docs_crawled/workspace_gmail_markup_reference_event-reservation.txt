이 유형은 이벤트에서 하나 이상의 참석자를 위한 예약을 선언하는 데 사용합니다. 티켓도 선언할 수 있습니다.
사용 사례
다음 사용 사례는 EventReservation 스키마가 사용되는 일반적인 예를 보여줍니다. 다음 예를 사용하여 마크업이 올바르게 구성되었는지 확인하세요.
참고: 시작하기 전에 이메일에 스키마를 삽입 [https://developers.google.com/workspace/gmail/markup/embedding-schemas-in-emails?hl=ko]하는 방법을 이해하고 스키마 테스트 [https://developers.google.com/workspace/gmail/markup/testing-your-schema?hl=ko]에 익숙해야 합니다.
티켓이 없는 기본 이벤트 알림
사용자의 이벤트 참석을 확인하는 이메일을 보내는 경우 다음 마크업을 포함하세요. 다음은 이메일을 EventReservation로 간주하기에 충분한 최소 마크업의 예입니다.
--- 탭: JSON-LD ---
<script type="application/ld+json">
{
  "@context": "http://schema.org",
  "@type": "EventReservation",
  "reservationNumber": "E123456789",
  "reservationStatus": "http://schema.org/Confirmed",
  "underName": {
    "@type": "Person",
    "name": "John Smith"
  },
  "reservationFor": {
    "@type": "Event",
    "name": "Foo Fighters Concert",
    "startDate": "2027-03-06T19:30:00-08:00",
    "location": {
      "@type": "Place",
      "name": "AT&T Park",
      "address": {
        "@type": "PostalAddress",
        "streetAddress": "24 Willie Mays Plaza",
        "addressLocality": "San Francisco",
        "addressRegion": "CA",
        "postalCode": "94107",
        "addressCountry": "US"
      }
    }
  }
}
</script>

티켓이 포함된 스포츠 또는 음악 이벤트
reservationFor 유형을 MusicEvent 또는 SportsEvent로 설정합니다. 이벤트가 MusicEvent인 경우 (예: 콘서트) performer.name 및 performer.image를 포함해야 합니다. 경쟁하는 두 팀 또는 선수가 있는 SportsEvent (예: 농구 경기)인 경우 두 팀을 공연자로 포함합니다.

티켓 및 지정석이 있는 이벤트
지정 좌석이 있는 단일 티켓의 경우 ticketToken, ticketNumber, venueSeat, venueRow, venueSection를 포함합니다. 좌석 위치를 설명하는 데 필요한 venueSeat, venueRow, venueSection 중 하나만 포함하면 됩니다.
numSeats는 포함하지 마세요.

여러 티켓

마크업 테스트
이메일 마크업 테스터 도구 [https://www.google.com/webmasters/markup-tester/?hl=ko]를 사용하여 마크업의 유효성을 검사할 수 있습니다. 마크업 코드를 붙여넣고 검증 버튼을 클릭하여 콘텐츠를 검사하고 있는 오류에 관한 보고서를 받습니다.
사양
이메일의 세부정보를 검토하여 이러한 속성이 이벤트 예약에 적용되는지 확인하세요. 이러한 추가 속성을 마크업하면 Google에서 사용자에게 훨씬 더 풍부한 이벤트 예약 설명을 표시할 수 있습니다.
EventReservation
유형 이름: EventReservation [https://developers.google.com/workspace/gmail/markup/reference/types/EventReservation?hl=ko]
예약 [https://developers.google.com/workspace/gmail/markup/reference/types/Reservation?hl=ko]을 확장합니다.
이름 유형 설명
작업 작업 [https://developers.google.com/workspace/gmail/markup/reference/types/Action?hl=ko] 이 항목에 대해 취할 수 있는 작업입니다.
action.name 텍스트 [https://developers.google.com/workspace/gmail/markup/reference/types/Text?hl=ko] 작업에 연결된 UI 요소에서 사용자에게 표시되는 문자열입니다.
action.url URL [https://developers.google.com/workspace/gmail/markup/reference/types/URL?hl=ko] 작업의 타겟 URL입니다. 명시적인 핸들러 필드가 제공되지 않으면 작업 핸들러가 이 URL을 WebActionHandler URL로 사용하여 WebActionHandler로 확장됩니다.
additionalTicketText 텍스트 [https://developers.google.com/workspace/gmail/markup/reference/types/Text?hl=ko] 티켓에 대한 추가 정보입니다.
