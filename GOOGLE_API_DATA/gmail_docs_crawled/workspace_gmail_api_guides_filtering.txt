검색 및 필터링 차이점: Gmail UI와 Gmail API [https://developers.google.com/workspace/gmail/api/guides/filtering?hl=ko#differences]
messages.list [https://developers.google.com/workspace/gmail/api/v1/reference/users/messages/list?hl=ko] 및 threads.list [https://developers.google.com/workspace/gmail/api/v1/reference/users/threads/list?hl=ko] 메서드를 사용하여 파일을 검색하거나 필터링할 수 있습니다. 이러한 메서드는 Gmail 웹 인터페이스와 동일한 고급 검색 구문 [https://support.google.com/mail/answer/7190?hl=ko]을 대부분 지원하는 q 매개변수를 허용합니다. Gmail UI와 Gmail API 간의 검색 및 필터 차이점 목록은 검색 필터 차이점: Gmail UI와 Gmail API 비교 [https://developers.google.com/workspace/gmail/api/guides/filtering?hl=ko#differences]를 참고하세요.
이 고급 구문을 사용하면 검색어를 사용하여 발신자, 날짜, 라벨과 같은 속성으로 메일을 필터링할 수 있습니다. 예를 들어 다음 쿼리는 2014년 1월에 사용자가 보낸 모든 메시지를 검색합니다.
GET https://www.googleapis.com/gmail/v1/users/me/messages?q=in:sent after:2014/01/01 before:2014/02/01
경고: 검색어에 사용된 모든 날짜는 PST 시간대의 해당 날짜 자정으로 해석됩니다. 다른 시간대의 정확한 날짜를 지정하려면 초 단위로 값을 전달하세요.
?q=in:sent after:1388552400 before:1391230800
검색어 외에도 labelIds 매개변수를 사용하여 라벨별로 메일과 스레드를 필터링할 수 있습니다. 이렇게 하면 지정된 시스템 또는 사용자 라벨이 적용된 메일과 대화목록을 검색할 수 있습니다. 자세한 내용은 messages.list [https://developers.google.com/workspace/gmail/api/v1/reference/users/messages/list?hl=ko] 또는 threads.list [https://developers.google.com/workspace/gmail/api/v1/reference/users/threads/list?hl=ko] 메서드 참조를 확인하세요.
검색 및 필터링 차이점: Gmail UI와 Gmail API
Gmail UI는 별칭 확장을 실행하여 Google Workspace 계정에서 계정 별칭을 추론할 수 있습니다. 예를 들어 myprimary@mycompany.net 계정이 있고 관리자가 해당 계정의 별칭을 myalias@mycompany.net로 설정한다고 가정해 보겠습니다. myalias@mycompany.net가 이메일을 보냈는데 'from: myprimary@mycompany.net)'를 검색하면 myalias@mycompany.net가 보낸 이메일이 Gmail UI의 검색 결과로 표시되지만 API 응답에는 표시되지 않습니다.
Gmail UI에서는 사용자가 스레드 전체 검색을 실행할 수 있지만 API에서는 실행할 수 없습니다.
