클라이언트 측 암호화 키 쌍을 사용 중지합니다. 인증된 사용자는 더 이상 키 쌍을 사용하여 수신 CSE 메일 텍스트를 복호화하거나 발신 CSE 메일에 서명할 수 없습니다.
액세스 권한을 다시 얻으려면 keypairs.enable [https://developers.google.com/workspace/gmail/api/reference/rest/v1/users.settings.cse.keypairs/enable?hl=ko#caribou.api.proto.MailboxService.EnableCseKeyPair]를 사용하여 키 쌍을 사용 설정합니다.
30일 후에는 keypairs.obliterate [https://developers.google.com/workspace/gmail/api/reference/rest/v1/users.settings.cse.keypairs/obliterate?hl=ko#caribou.api.proto.MailboxService.ObliterateCseKeyPair] 메서드를 사용하여 키 쌍을 영구 삭제할 수 있습니다.
조직의 사용자에 대한 ID 및 키 쌍을 관리하는 관리자의 경우 요청에 https://www.googleapis.com/auth/gmail.settings.basic 범위로 사용자를 가장할 수 있는 도메인 전체 위임 권한 [https://developers.google.com/identity/protocols/OAuth2ServiceAccount?hl=ko#delegatingauthority]이 있는 서비스 계정 [https://developers.google.com/identity/protocols/OAuth2ServiceAccount?hl=ko]으로 승인해야 합니다.
자체 ID 및 키 쌍을 관리하는 사용자의 경우 요청에 하드웨어 키 암호화 [https://support.google.com/a/answer/14153163?hl=ko]가 사용 설정되고 구성되어 있어야 합니다.
HTTP 요청
POST https://gmail.googleapis.com/gmail/v1/users/{userId}/settings/cse/keypairs/{keyPairId}:disable
URL은 gRPC 트랜스코딩 [https://google.aip.dev/127] 문법을 사용합니다.
