작업을 사용하면 사용자가 Gmail 내에서 바로 제품 또는 서비스와 상호작용할 수 있습니다.
예를 들어 잡지 정기 결제가 만료된다는 알림을 보낼 때 알림에서 바로 갱신 옵션을 표시할 수 있습니다.
작업은 인앱 액션 또는 이동 작업으로 두 가지 방법으로 선언할 수 있습니다. 현재 여러 유형의 작업이 지원됩니다.
인앱 액션 수
인앱 액션은 사용자를 다른 웹사이트로 보내지 않고 Gmail 내에서 바로 처리됩니다. 이러한 작업에는 원클릭 작업이 포함됩니다.
원클릭 작업
사용자가 사전 정의된 요청을 확인하는 것이 예상되는 동작인 시나리오가 많습니다. 예를 들어 사용자가 사이트에 등록하면 등록을 확인하라는 이메일을 받습니다. 마찬가지로 영화 또는 음악 추천 사이트에서 나중에 즐길 수 있도록 사용자가 항목을 대기열에 추가하도록 유도할 수 있습니다.

Gmail의 원클릭 작업
자세한 내용은 원클릭 작업 참조 [https://developers.google.com/workspace/gmail/markup/reference/one-click-action?hl=ko]를 참고하세요.
바로가기 작업
더 복잡한 상호작용의 경우 Go-To Actions를 사용하여 작업을 실행할 수 있는 페이지로 바로 연결되는 링크를 제공할 수 있습니다. 버튼을 클릭하면 사용자가 작업 정의에 지정된 페이지로 리디렉션됩니다.
바로가기 작업은 여러 단계가 필요하거나 받은편지함에서 실행할 수 없는 중요한 데이터 수집이 필요한 작업에 사용해야 합니다. 예를 들어 체크인 알림을 보내는 항공사는 이동 작업을 추가하여 사용자가 항공사 웹사이트로 빠르게 이동하여 좌석을 선택하거나 업그레이드를 요청할 수 있습니다.
