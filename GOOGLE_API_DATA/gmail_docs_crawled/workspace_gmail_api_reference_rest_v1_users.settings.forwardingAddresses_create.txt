전달 주소를 만듭니다. 소유권 확인이 필요한 경우 수신자에게 메시지가 전송되고 리소스의 인증 상태가 pending로 설정됩니다. 그렇지 않으면 인증 상태가 accepted로 설정된 리소스가 생성됩니다.
이 메서드는 도메인 전체 권한이 위임된 서비스 계정 클라이언트에서만 사용할 수 있습니다.
HTTP 요청
POST https://gmail.googleapis.com/gmail/v1/users/{userId}/settings/forwardingAddresses
URL은 gRPC 트랜스코딩 [https://google.aip.dev/127] 문법을 사용합니다.
경로 매개변수
매개변수
userId
string
사용자의 이메일 주소 특수 값 'me'를 사용하여 인증된 사용자를 나타낼 수 있습니다.
요청 본문
요청 본문에 ForwardingAddress [https://developers.google.com/workspace/gmail/api/reference/rest/v1/users.settings.forwardingAddresses?hl=ko#ForwardingAddress]의 인스턴스가 포함됩니다.
응답 본문
성공한 경우 응답 본문에 새로 생성된 ForwardingAddress [https://developers.google.com/workspace/gmail/api/reference/rest/v1/users.settings.forwardingAddresses?hl=ko#ForwardingAddress]의 인스턴스가 포함됩니다.
승인 범위
다음 OAuth 범위가 필요합니다.
https://www.googleapis.com/auth/gmail.settings.sharing
자세한 내용은 승인 가이드 [https://developers.google.com/workspace/guides/configure-oauth-consent?hl=ko]를 참고하세요.
