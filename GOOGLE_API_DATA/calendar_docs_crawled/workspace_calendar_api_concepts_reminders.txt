Calendar API는 리마인더와 알림을 지원합니다.
미리 알림은 일정이 시작되기 전 지정된 시간에 트리거되는 알람입니다.
알림을 통해 사용자는 캘린더의 일정 변경사항을 확인할 수 있습니다.
후자는 사용자에게 전송되는 대신 다른 서버에 캘린더 변경사항을 알리는 푸시 알림 [https://developers.google.com/workspace/calendar/v3/push?hl=ko]과 혼동해서는 안 됩니다.
Google Calendar의 알림에 관한 자세한 내용은 Google Calendar 알림 수정하기 [https://support.google.com/calendar/answer/37242?hl=ko]를 참고하세요.
리마인더
리마인더는 다음으로 구성됩니다.
알림을 표시할 시점(이벤트 시작 시간 전의 분)
사용할 전송 방법 (전송 메커니즘 [https://developers.google.com/workspace/calendar/concepts/reminders?hl=ko#delivery_mechanisms] 참고)
리마인더는 전체 캘린더와 개별 일정에 대해 지정할 수 있습니다. 사용자는 각 캘린더에 기본 리마인더를 설정할 수 있으며, 이러한 기본값은 해당 캘린더 내의 모든 일정에 적용됩니다. 하지만 사용자는 개별 일정의 기본값을 재정의하여 다른 알림 집합으로 바꿀 수도 있습니다.
기본 리마인더
리마인더는 인증된 사용자에게만 해당하는 비공개 정보이며 여러 사용자 간에 공유되지 않습니다. 결과:
기본 알림은 사용자별 캘린더 메타데이터가 포함된 CalendarList 컬렉션을 통해 조작됩니다.
모든 사용자 간에 공유되는 전역 메타데이터가 포함된 캘린더 컬렉션을 통해 액세스할 수 없습니다.
이벤트 목록 쿼리를 실행할 때 기본 알림도 반환됩니다.
기본 리마인더 재정의
이벤트를 삽입하거나 수정할 때 기본 알림을 재정의하려면 reminders.useDefault [https://developers.google.com/workspace/calendar/v3/reference/events?hl=ko#reminders.useDefault]를 false로 설정하고 reminders.overrides [https://developers.google.com/workspace/calendar/v3/reference/events?hl=ko#reminders.overrides]를 새 알림 세트로 채웁니다.
"reminders": {
  "useDefault": false,
  # Overrides can be set if and only if useDefault is false.
  "overrides": [
      {
        "method": "reminderMethod",
        "minutes": "reminderMinutes"
      },
      # ...
  ]
}
기본 알림 세트로 되돌리려면 reminders.useDefault [https://developers.google.com/workspace/calendar/v3/reference/events?hl=ko#reminders.useDefault] 설정을 true로 업데이트합니다.
알림
Calendar는 다음 알림 유형을 지원합니다.
일정 생성: 사용자의 캘린더 중 하나에 새 일정이 추가됩니다.
일정 변경: 주최자가 사용자가 초대된 일정을 수정했습니다.
일정 취소: 사용자가 초대된 일정이 취소됩니다.
참석자 응답: 사용자가 만든 일정의 참석자가 응답 상태를 변경했습니다.
일정: 하루가 시작될 때 전송되는 사용자 캘린더의 모든 일정 목록입니다.
사용자는 캘린더별로 사용 설정할 알림과 각 알림 유형의 전송 방법을 결정할 수 있습니다. 이 설정은 다른 사용자와 공유되지 않습니다. 기본 리마인더와 마찬가지로 CalendarList 컬렉션을 통해 액세스할 수 있습니다.
API로 삽입되거나 업데이트된 이벤트에 대해 참석자에게 이메일 알림을 보내려면 insert [https://developers.google.com/workspace/calendar/api/v3/reference/events/insert?hl=ko] 또는 update [https://developers.google.com/workspace/calendar/api/v3/reference/events/update?hl=ko] 메서드를 호출하고 sendUpdates 매개변수를 "all" 또는 "externalOnly"로 설정합니다.
참고: Google 이외의 이메일 주소를 사용하는 참석자의 경우 이러한 알림 이메일이 이벤트에 대해 알 수 있는 유일한 방법이며, 캘린더에 자동으로 추가되지는 않습니다.
