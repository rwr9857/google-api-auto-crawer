도메인 리소스 및 회의실 [https://developers.google.com/workspace/calendar/api/concepts/domain?hl=ko#domain_resources_rooms]
앱으로 도메인 캘린더에 액세스 [https://developers.google.com/workspace/calendar/api/concepts/domain?hl=ko#accessing_domain_calendars_as_an_app]
Calendar API는 주로 Google Workspace [https://workspace.google.com/?hl=ko] 고객에게 유용한 여러 개념을 제공합니다. 이러한 고객은 특정 이벤트를 위해 예약할 수 있는 리소스(회의실, 프로젝터 등)를 보유하고 있는 경우가 많습니다. 또한 도메인의 모든 사용자 캘린더에 액세스해야 하는 내부 앱을 빌드하는 것이 일반적입니다(예: 회사 일정을 추가하는 경우).
도메인 리소스 및 회의실
이메일 주소를 사용하여 참석자로 일정에 리소스와 회의실을 추가하여 예약할 수 있습니다. 초대를 받으면 초대 사용자의 일정 및 액세스 권한에 따라 자동으로 일정을 수락하거나 거부합니다.
참고: Calendar API는 리소스 캘린더를 만드는 방법을 제공하지 않습니다. 이렇게 하려면 Directory API의 Calendar Resource [https://developers.google.com/workspace/admin/directory/reference/rest/v1/resources.calendars?hl=ko] 객체를 사용해야 합니다. calendarId는 캘린더 리소스 표현의 resourceEmail 필드에 해당합니다.
앱으로 도메인 캘린더에 액세스
앱이 서비스 계정 [https://developers.google.com/identity/protocols/OAuth2ServiceAccount?hl=ko]을 사용하여 인증하는 경우 사용자 인증 정보 없이 도메인 소유 캘린더에 액세스할 수 있습니다. 서비스 계정에는 도메인 전체 권한 위임 [https://developers.google.com/identity/protocols/OAuth2ServiceAccount?hl=ko#delegatingauthority]을 사용하여 필요한 액세스 권한이 있어야 합니다. 사용자 계정을 가장하려면 GoogleCredential 팩토리의 setServiceAccountUser 메서드로 사용자 계정의 이메일 주소를 지정합니다.
